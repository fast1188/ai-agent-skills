# write-email

> 中文场景邮件生成器 —— 商务 / 求职 / 催办 / 道歉 / 感谢 5 大场景
>
> 生成的邮件**不像 AI 写**：短句、有节奏、有具体行动、零客套

## 这是什么

`write-email` 把"我想发一封邮件"这种模糊需求，转化成可直接发的成稿。
不是"模板套娃"——而是**分析场景 → 选结构 → 填血肉**。

5 大场景：

| 场景 | 触发关键词 | 长度 |
|------|-----------|------|
| 商务 BD | 客户/合作/报价/提案 | 150-250 字 |
| 求职 | 应聘/简历/面试 | 200-300 字 |
| 催办 | 跟进/进度/卡在哪 | 80-150 字 |
| 道歉 | 抱歉/事故/延期 | 100-200 字 |
| 感谢 | 谢谢/帮了大忙 | 60-120 字 |

## 输出格式

```
主题行: [≤18 字，吸引力 + 具体]

[称谓],
[正文 3-6 段，每段 1-2 句，不超 3 行]

[落款]
[署名]
```

## 使用

**在 Claude Code 中：**
```
你: 用 write-email skill 给客户王总写一封 SaaS 试用邀请
[自动调用 skill]
AI: 输出完整邮件稿
```

**单独 CLI：**
```bash
py write_email.py "给客户王总写 SaaS 试用邀请" --to "王总" --tone 商务
```

## 设计原则（不像 AI 的 6 条铁律）

1. **短句优先** — 一句不超过 20 字
2. **具体数字** — 不"尽快"，要"周三前"
3. **零客套** — 不"打扰了/冒昧/敬请"
4. **明确 CTA** — 每封邮件结尾一句话告诉对方做什么
5. **有节奏** — 短长长短，避免每段一样长
6. **署名真实** — 用具体名字 + 职位，不"敬上"

## 例子

### 例 1：商务 BD

**输入**："给某 SaaS 公司 CTO 写试用邀请"

**输出**：
```
主题行: 14 天免费试用,帮贵司 AI 成本省 60%

李总,

我看了贵司最近发的工程博客,AI 调用这块成本应该不低。
我们的多 AI 路由(free-ai-router) 跑了 3 个月,帮 50+ 客户
平均省 40%。

不占用您时间,给个 14 天试用密钥,跑一跑你们实际流量就行。

老张
free-ai-router 创始人
```

### 例 2：催办

**输入**："催问客户上个月提的需求什么时候能排期"

**输出**：
```
主题行: 上次提的 XX 集成,现在卡在哪了?

王老师,

您 5/20 提的 XX 集成需求,目前还在待评估。
方便的话,周三前能给个时间预期吗?
我们这边要排下季度资源。

老张
```

### 例 3：道歉（事故）

**输入**："昨天 API 挂了 2 小时,给所有付费用户发道歉信"

**输出**：
```
主题行: 昨天事故说明 + 补偿方案

各位付费用户,

昨天 14:00-16:00 API 挂了 2 小时。
原因: MiniMax 官方限流,我们的 fallback 路由没顶住。
补偿: 本月所有付费用户 +7 天免费。
改进: 今晚上线二级 fallback,本周完成压测。

老张
free-ai-router 团队
```

## 6 条反 AI 检测要点

1. ❌ "希望对您有所帮助" → ✅ "省 40%"
2. ❌ "感谢您的宝贵时间" → ✅ "不占用您时间"
3. ❌ "如有任何疑问请随时联系" → ✅ "周三前能给个时间预期吗"
4. ❌ "我们致力于提供..." → ✅ "跑了 3 个月"
5. ❌ "祝商祺" → ✅ 直接署名
6. ❌ "亲爱的合作伙伴" → ✅ "李总,"
